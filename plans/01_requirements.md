## 3. 유스케이스 (Use Cases)

### UC-001: C 개념 질문하기
```
Actor: 학습자
Precondition: 웹사이트 접속
Flow:
  1. 학습자가 AI 튜터 채팅창에 질문 입력
  2. 시스템이 AI에게 질문 전달
  3. AI가 답변 생성
  4. 시스템이 채팅창에 답변 표시
Postcondition: 학습자가 답변 확인
```

### UC-002: C 코드 실행하기
```
Actor: 학습자
Precondition: 웹사이트 접속
Flow:
  1. 학습자가 코드 에디터에 C 코드 작성
  2. 학습자가 "실행" 버튼 클릭
  3. 시스템이 코드를 백엔드로 전송
  4. 백엔드가 Docker에서 컴파일/실행
  5. 시스템이 결과를 화면에 표시
Alternative:
  4a. 컴파일 에러 발생 → 에러 메시지 표시
  4b. 타임아웃 발생 → 타임아웃 메시지 표시
Postcondition: 학습자가 실행 결과 확인
```

### UC-003: 메모리 할당 시각화 보기
```
Actor: 학습자
Precondition: 웹사이트 접속
Flow:
  1. 학습자가 메모리 시뮬레이터 탭 선택
  2. 학습자가 malloc 크기 입력
  3. 시스템이 메모리 상태 계산
  4. 시스템이 애니메이션으로 할당 과정 표시
Postcondition: 학습자가 메모리 상태 확인
```

## 4. 제약사항 (Constraints)

| 항목 | 제약 |
|------|------|
| 실행 언어 | C만 지원 (C++ 미지원) |
| 컴파일러 | gcc만 사용 |
| 외부 라이브러리 | 표준 라이브러리만 허용 |

## 5. 가정사항 (Assumptions)

1. 사용자는 기본적인 프로그래밍 개념을 알고 있음
2. 브라우저에서 JavaScript가 활성화되어 있음
3. 서버에 Docker가 설치되어 있음
4. 로컬 LLM 서버(Ollama)가 실행 중임
